Output: New visualStore.ts with VisualState interface, vanilla Zustand store instance, and typed accessor hooks.

  <name>Task 1: Create visualStore.ts with vanilla Zustand store</name>
  <files>apps/client/src/store/visualStore.ts</files>
  <action>
Create `apps/client/src/store/visualStore.ts` with:

1. Import `createStore` from `zustand/vanilla` (NOT the default Zustand create)
2. Import THREE.js types for Vector3
3. Define VisualState interface with:
   - `playerPositions: Map<string, { x: number; y: number; z: number }>` - Target positions for interpolation
   - `playerRotations: Map<string, number>` - Player lookYaw values
   - `cameraShake: { intensity: number; time: number }` - Camera shake intensity and remaining time
   - `slideFov: number` - FOV adjustment during slide
   - `interpolationTargets: Map<string, { x: number; y: number; z: number }>` - Raw server positions before interpolation

4. Create initialVisualState with empty Maps and zero values

5. Export `visualStore` using `createStore<VisualState>(() => initialVisualState)`

6. Export convenience functions for non-reactive updates:
   - `setPlayerVisualPosition(playerId: string, position: { x: number; y: number; z: number })` - Updates playerPositions Map
   - `setPlayerVisualRotation(playerId: string, rotation: number)` - Updates playerRotations Map
   - `setCameraShake(intensity: number, time: number)` - Updates camera shake
   - `setSlideFov(fov: number)` - Updates slide FOV

7. Export typed hook `useVisualStore<T>(selector: (state: VisualState) => T)` using `useStore` from 'zustand' for optional reactive access

DO NOT use default Zustand `create()` - must use vanilla `createStore()` to avoid React re-renders on mutation.
