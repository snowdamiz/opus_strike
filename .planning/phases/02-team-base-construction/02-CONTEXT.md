## Phase Boundary

Build both team bases with spawn points and flag capture zones integrated with game systems. Team A has tech/platform aesthetic (from Phase 1), Team B has natural/cave aesthetic. Players spawn here, flags live here, captures happen here. Cover elements and combat positions are Phase 3.

</domain>

<decisions>
## Implementation Decisions

### Spawn Point Layout
- Distributed spawns — multiple spawn positions spread across each base
- Semi-protected areas — some cover nearby but not fully enclosed spawn rooms
- Subtle spawn indicators — floor texture change or faint glow, noticeable if looking but not dominant

### Flag Zone Design
- All entry points equal access — no preferred approach, each route roughly equivalent
- Defender advantage — cover and sightlines favor defenders, flag is hard to grab
- Warning indicators when contested — zone glows or pulses differently when enemies nearby

### Game Integration
- Dramatic flag pickup effects — flash, particle burst, sound when flag is grabbed
- Visible carrier marker — flag carrier has outline, icon, or visible flag model
- Quick capture confirmation — score updates, brief audio, game flow continues smoothly
- Manual flag return only — defenders must touch dropped flag to return it (no auto-return)

### Claude's Discretion
- Base visual identity strength (strongly vs subtly themed)
- Landmark decisions (separate landmarks vs flag zone as focal point)
- Base boundary treatment (clear markers vs gradual transition)
- Flag presentation style (pedestal vs ground level)
- Spawn point count per team
- Visual style references (gameplay-first vs detailed environments)

</decisions>
